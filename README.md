# agent-manager-workflow — AI 总经理工作流

> 一键安装，即刻拥有一个懂调度、会审查、能持久化的 AI 总经理团队。

## 这是什么？

你是**老板**，Claude 是你的**总经理**，总经理下面有一群**员工**：

```
你（老板）
  │
  └── 总经理（任务调度、质量把控、向你汇报）
        │
        ├── 研究员   —— 查资料、做调研
        ├── 写作者   —— 写代码、写文档
        ├── 审查员   —— 挑毛病、查安全
        ├── 部署者   —— 处置交付单（独立调用）
        └── 策展人   —— 维护知识库、发现关联
```

你说一句话，总经理走完整 6 阶段流水线，每阶段确认后才推进。

- **1 个总经理**：负责任务调度、需求拆解、质量把控
- **5 个员工 Agent**：研究员、写作者、审查员、部署者、策展人
- **8 个中文命令**：/交办 /快办 /审查 /存档 /校准 /近况 /脑暴 /update

## v2.0 新特性

- **6 阶段固定流程**：需求澄清 → 技术方案 → Spec 撰写 → 代码实现 → 代码审查 → 交付通知，每阶段 ⛔ 确认门禁
- **用户级配置**：`~/.claude/agent-manager/` 存放流程、偏好、决策、校准，换项目跟你走
- **流程可覆盖**：项目级 `.claude/guidance/TASK_PIPELINE.md` 优先于用户级
- **`/update` 命令**：插件升级时对比差异，支持全部覆盖/保持/逐项/自定义
- **流程执行状态追踪**：每个任务维护 `流程执行状态.json`，中断后可续接
- **偏好自动积累**：manager 自动识别你的长期偏好并记录

## 快速开始

在终端运行：

```bash
claude plugin marketplace add https://github.com/miaosong-z/agent-manager-workflow.git
claude plugin install agent-manager-workflow
```

安装后在对话中即可使用：

```
/交办 "帮我调研 MCP 协议并写一篇笔记"
/审查 src/main.py
/近况
```

首次 `/交办` 会自动初始化 `~/.claude/agent-manager/`，包含默认流程和用户配置。

## 如何更新

在终端运行：

```bash
claude plugin update agent-manager-workflow
```

更新后重启 Claude Code，首次运行 `/交办` 会检测到版本变化并提醒：

```
插件已更新到 v2.0.0，你的配置基于 v1.0.0。运行 /update 查看更新内容。
```

此时运行 `/update`，会展示新旧流程的差异，你可以选择：
- **全部覆盖** — 使用新版流程
- **保持当前** — 继续用你优化过的流程，不再提醒
- **逐项确认** — 逐个文件决定
- **自定义流程** — 自己设计工作流

## 命令详解

### /交办 — 主入口
提交任务，走完整 6 阶段流程。不判断复杂度，所有任务统一处理。
```
/交办 "写一篇关于 AI Agent 协作模式的深度笔记"
/交办 "重构这个模块，提高性能"
```

### /快办 — 快速处理
简单任务不走完整管道，直接产出。
```
/快办 "把这个拼写错误改掉"
```

### /审查 — 质量检查
拉审查员出来单独审计指定文件。
```
/审查 src/main.py
```

### /存档 — 持久化
将会话中有价值的内容保存到 memory 和 guidance。
```
/存档
```

### /校准 — 权限调整
回顾决策，调整哪些事以后可以自己做主。
```
/校准
```

### /近况 — 全局视图
展示项目仪表盘。
```
/近况
```

### /脑暴 — 发散讨论
纯思想实验，不写代码、不出方案、不做决策。适合聊想法、梳理想象、探索可能性。
```
/脑暴 "AI agent 的记忆系统可以有哪些设计思路？"
```

### /update — 升级检查
对比插件新版本与用户当前配置的差异，选择覆盖、保持、逐项确认或自定义流程。
```
/update
```

## 工作流

```
用户说一句话
  → /交办 创建任务
    → 阶段1：需求澄清   ⛔ 确认
    → 阶段2：技术方案   ⛔ 确认
    → 阶段3：Spec 撰写  ⛔ 确认（缺 OpenSpec → 按需安装）
    → 阶段4：代码实现
    → 阶段5：代码审查   ⛔ 确认（多轮循环）
    → 阶段6：交付通知
```

每阶段产出存入 `.claude/tasks/<task-name>/`，`流程执行状态.json` 实时追踪进度。

## 用户级 vs 项目级

| 层级 | 位置 | 存放内容 |
|------|------|---------|
| 用户级 | `~/.claude/agent-manager/` | 默认流程、决策记录、偏好、校准 |
| 项目级 | `.claude/guidance/` | 架构约定、指导索引、可选流程覆盖 |
| 项目级 | `.claude/tasks/<task-name>/` | 任务阶段文档、流程执行状态 |

流程优先级：项目级 > 用户级 > 插件默认。对话中优化流程时，manager 会询问"当前项目 or 全部项目"。

## 子 Agent 团队

| Agent | 模型 | 职责 |
|-------|------|------|
| researcher | sonnet | 深度调研、查资料、标注来源 |
| writer | sonnet | 撰写代码和文档、产出成品 |
| reviewer | sonnet | 挑毛病、查安全、验证逻辑 |
| deployer | sonnet | 话题终结者，接收交付单、确认后执行（独立调用） |
| curator | haiku | 知识关联、索引维护 |

> 架构师角色使用内置 Plan agent 类型（通过 Agent tool 指定 subagent_type: 'Plan'），不需要独立的 planner agent 文件。

## 决策分级

| 级别 | 范围 | 处理 |
|------|------|------|
| A 级 | 方向、取舍、正确性 | 必须上报用户，记录到 decisions/ |
| B 级 | 常规协调 | 校准后授权 |
| C 级 | 执行细节 | 自行决定 |

## 进阶用法：让 Claude 成为常驻总经理

### 默认模式 vs 进阶模式

**默认模式（不复制模板）**——你是老板，需要时叫总经理过来：

```
你说：/交办 "帮我写一篇 Redis 持久化笔记"
       → Claude 临时切换为总经理模式
       → 走 6 阶段流程，每阶段确认
       → 任务完成，Claude 回到普通模式

你说："今天天气怎么样？"
       → 普通 Claude，直接回答，不走管道
```

你需要总经理时才叫他，平常 Claude 就是 Claude。

**进阶模式（复制 CLAUDE.md 模板）**——Claude 永久扮演总经理：

```bash
cp ${CLAUDE_PLUGIN_ROOT}/templates/CLAUDE.md.template ./CLAUDE.local.md
```

或者更简单的方式：
```
/交办 "帮我初始化总经理角色，写入 CLAUDE.local.md"
```

效果：

```
你说："帮我写一篇 Redis 持久化笔记"
       → Claude 自动识别这是工作任务
       → 走 6 阶段流程，每阶段向你确认
       → 汇总汇报，提醒你存档

你说："这个方案用 A 还是 B？"
       → Claude 识别这是方向性决策（A 级）
       → 给出分析，但等你拍板，不自己选

你说："以后方案阶段不需要我确认了"
       → Claude 识别这是流程优化请求
       → 问"当前项目生效还是所有项目？"
       → 按你的选择写入对应位置
```

**一句话总结**：默认模式是"按需叫总经理"，进阶模式是"Claude 就是总经理"。后者不需要打 `/交办`，说话就行。

### 进阶模式的代价

- Claude 会变得更有"管理感"——更多汇报、更多确认、更结构化。闲聊也可能被当成任务分析。
- 如果你只是想快速问个问题，偶尔需要说"别走流程，直接回答"。
- 建议在**主力工作项目**中启用，临时项目保持默认模式。

## 常见问题

**Q: 中文命令在非中文环境下能用吗？**
A: 可以。插件安装后，命令命名空间为 /agent-manager-workflow:交办，无冲突时可直接用 /交办。

**Q: 可以只用部分功能吗？**
A: 可以。每个 Skill 和 Agent 都是独立的。你可以只用 /审查，不用 /交办。

**Q: 能自定义子 Agent 吗？**
A: 可以。编辑 `${CLAUDE_PLUGIN_ROOT}/agents/*.md` 调整角色提示、模型、工具权限。

**Q: 能修改流程吗？**
A: 可以。对话中说"以后 XX 阶段不需要确认了"，manager 会自动识别并询问生效范围。也可通过 `/update` 在插件升级时自定义流程。

**Q: 换项目后偏好还在吗？**
A: 在。用户级配置（流程、偏好、决策）存于 `~/.claude/agent-manager/`，跟人不跟项目。
