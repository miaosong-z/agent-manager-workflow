# agent-manager-workflow — AI 总经理工作流

> 一键安装，即刻拥有一个懂调度、会审查、能持久化的 AI 总经理团队。

## 这是什么？

你是**老板**，Claude 是你的**总经理**，总经理下面有一群**员工**：

```
你（老板）
  │
  └── 总经理（任务调度、质量把控、向你汇报）
        │
        ├── 研究员   —— 查资料、做调研
        ├── 写作者   —— 写代码、写文档
        ├── 审查员   —— 挑毛病、查安全
        └── 策展人   —— 维护知识库、发现关联
```

你只需要说一句话，总经理自动判断该找谁、怎么做、要不要先确认再执行。

- **1 个总经理**：负责任务调度、需求拆解、质量把控
- **4 个员工 Agent**：研究员、写作者、审查员、策展人
- **6 个中文命令**：/交办 /快办 /审查 /存档 /校准 /近况

## 快速开始

```bash
# 添加 marketplace 并安装
/plugin marketplace add miaosong-z/agent-manager-workflow
/plugin install agent-manager-workflow
```

安装完成即可使用：
```
/交办 "帮我调研 MCP 协议并写一篇笔记"
/审查 src/main.py
/近况
```

不需要额外配置。

## 命令详解

### /交办 — 主入口
提交任务，自动判断复杂度并调度合适的子 Agent。
```
/交办 "写一篇关于 AI Agent 协作模式的深度笔记"
/交办 "重构这个模块，提高性能"
```

### /快办 — 快速处理
简单任务不走完整管道，直接产出。
```
/快办 "把这个拼写错误改掉"
```

### /审查 — 质量检查
拉审查员出来单独审计指定文件。
```
/审查 src/main.py
```

### /存档 — 持久化
将会话中有价值的内容保存到 memory 和 guidance。
```
/存档
```

### /校准 — 权限调整
回顾决策，调整哪些事以后可以自己做主。
```
/校准
```

### /近况 — 全局视图
展示项目仪表盘。
```
/近况
```

## 工作流

```
用户说一句话 
  → /交办 判断复杂度
    → 简单：直接 spawn writer 产出
    → 中等：spawn researcher 调研 → writer 撰写 → reviewer 审查 → 汇报
    → 复杂：调研 → 方案设计 → 用户确认 → 实施 → 审查 → 汇报
```

## 子 Agent 团队

| Agent | 模型 | 职责 |
|-------|------|------|
| researcher | sonnet | 深度调研、查资料、标注来源 |
| writer | sonnet | 撰写代码和文档、产出成品 |
| reviewer | sonnet | 挑毛病、查安全、验证逻辑 |
| curator | haiku | 知识关联、索引维护 |

> 架构师角色使用内置 Plan agent 类型（通过 Agent tool 指定 subagent_type: 'Plan'），不需要独立的 planner agent 文件。

## 决策分级

| 级别 | 范围 | 处理 |
|------|------|------|
| A 级 | 方向、取舍、正确性 | 必须上报用户 |
| B 级 | 常规协调 | 校准后授权 |
| C 级 | 执行细节 | 自行决定 |

## 进阶用法：让 Claude 成为常驻总经理

### 默认模式 vs 进阶模式

**默认模式（不复制模板）**——你是老板，需要时叫总经理过来：

```
你说：/交办 "帮我写一篇 Redis 持久化笔记"
       → Claude 临时切换为总经理模式
       → 判断任务复杂度、调度员工、审查、汇报
       → 任务完成，Claude 回到普通模式

你说："今天天气怎么样？"
       → 普通 Claude，直接回答，不走管道
```

你需要总经理时才叫他，平常 Claude 就是 Claude。

**进阶模式（复制 CLAUDE.md 模板）**——Claude 永久扮演总经理：

```bash
cp ${CLAUDE_PLUGIN_ROOT}/templates/CLAUDE.md.template ./CLAUDE.md
```

效果：

```
你说："帮我写一篇 Redis 持久化笔记"
       → Claude 自动识别这是工作任务
       → 自动判断复杂度（中等）
       → 自动 spawn 研究员调研 → 写作者撰写 → 审查员审查
       → 汇总汇报，提醒你存档

你说："改一下 README 里的拼写"
       → Claude 自动识别这是简单任务
       → 直接 spawn 写作者改好给你

你说："这个方案用 A 还是 B？"
       → Claude 识别这是方向性决策（A 级）
       → 给出分析，但等你拍板，不自己选

你说："今天就到这"
       → Claude 提醒你 /存档，把今天的产出持久化
```

**一句话总结**：默认模式是"按需叫总经理"，进阶模式是"Claude 就是总经理"。后者不需要打 `/交办`，说话就行。

### 进阶模式的代价

- Claude 会变得更有"管理感"——更多汇报、更多确认、更结构化。闲聊也可能被当成任务分析。
- 如果你只是想快速问个问题，偶尔需要说"别走流程，直接回答"。
- 建议在**主力工作项目**中启用，临时项目保持默认模式。

## 常见问题

**Q: 中文命令在非中文环境下能用吗？**
A: 可以。插件安装后，命令命名空间为 /agent-manager-workflow:交办，无冲突时可直接用 /交办。

**Q: 可以只用部分功能吗？**
A: 可以。每个 Skill 和 Agent 都是独立的。你可以只用 /审查，不用 /交办。

**Q: 能自定义子 Agent 吗？**
A: 可以。编辑 `${CLAUDE_PLUGIN_ROOT}/agents/*.md` 调整角色提示、模型、工具权限。
